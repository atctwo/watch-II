22 /*
23  * The time update takes place in the sntp_sync_time() function.
24  * The user has the ability to redefine this function in order
25  * to re-define its functionality. This function has two time update modes,
26  * which can be set via the sntp_set_sync_mode() function.
27  * Two modes are available:
28  * - the first is an immediate update when receiving time from the sntp server,
29  * - the second is a smooth time update (if the time error is no more than 35 minutes,
30  * and an immediate update if the error is more than 35 minutes).
31  *
32  * To receive notification of time synchronization,
33  * you can use the callback function or get the synchronization status
34  * via the sntp_get_sync_status() function.
35  *
36  * To determine the time synchronization time on the device, you can use:
37  * 1) sntp_set_time_sync_notification_cb() function to set the callback function,
38  * which is convenient to use to receive notification of the update time.
39  * 2) sntp_get_sync_status() function for getting time synchronization status.
40  * After the time synchronization is completed, the status will be
41  * SNTP_SYNC_STATUS_COMPLETED, after, it will be reseted to SNTP_SYNC_STATUS_RESET
42  * to wait for the next sync cycle.
43  */
44
